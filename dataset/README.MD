This folder contains the complete pipeline for creating the Elara Dataset. It is designed to be modular and reusable for any educational subject within the Egyptian curriculum.

 Scripts Overview
1. dataset_generator.py (The Engine)
This is a generic, highly configurable synthetic data generator powered by the Gemini API.

Purpose: Generates realistic, long-form Arabic chat transcripts between a tutor (AI) and a student.

Key Feature: Features a "Configuration Section" at the top. You can swap subjects (e.g., from Statics to Physics or Arabic) without touching the core API logic.

Output: Generates raw .jsonl files where each line represents a full conversation and its corresponding analytical report.

2. merge_dataset.py (The Aggregator)
Since data generation is often done in batches or for different subjects (e.g., Statics & Dynamics & arabic), this script handles the merging process.

Purpose: Combines multiple small .jsonl files into one master dataset.

Integrity: Ensures all entries follow the correct nested JSON structure required by the training notebooks.

3. data_cleaning.py (The Refiner)
A crucial step for a Data Scientist to ensure data quality. This script prepares the raw data for the Qwen model.

Deduplication: Removes any duplicate conversations generated by the API.

Feature Engineering: As per project requirements, it removes functional timestamps from the student-AI interactions to keep the focus on pedagogical content.

Validation: Checks that the Arabic text is encoded correctly (UTF-8) and that the system, user, and assistant roles are properly balanced.
